## DSA Chunk C — Sliding Window Visualizer (Maximum Sum Subarray) — 2026-07-24

### What was built
- **Sliding Window Trace Generator**: A Python script (`generate_sliding_window_trace.py`) was created to generate `trace.json` for the "Maximum Sum Subarray of Size K" problem. This script outputs a `trace.json` file following the `DSATrace` schema, including steps for an optimized (sliding window) solution.
- **Sliding Window Visualizer Component**: A new React component (`app/dsa/flagship/max-sum-subarray/page.tsx`) was implemented to visually represent the execution of the Maximum Sum Subarray problem using the sliding window technique. This component consumes the generated `trace.json`.

### Decisions made (and why)
- **Architectural Consistency**: The visualizer component was built to mirror the architecture and patterns established by the existing Two Sum visualizer, including the use of `fetchDSATrace` from `shared/lib/dsa-sync.ts` to fetch trace data.
- **Trace Schema Adherence**: The `trace.json` generated by the Python script strictly adheres to the `DSATrace` interface defined in `shared/lib/dsa-sync.ts`, ensuring compatibility with the visualizer.
- **Interactive Controls**: Implemented play/pause, step-forward, step-back, and reset controls for interactive exploration of the algorithm steps.
- **Accessibility**: Integrated `useReducedMotion` to respect user preferences for reduced motion, providing a step-by-step click mode instead of auto-play when enabled.
- **Styling**: Utilized existing Tailwind CSS design tokens for a consistent look and feel.
- **Hardcoded Example**: For demonstration purposes, the input array (`nums = [2, 1, 5, 1, 3, 2]`) and window size (`k = 3`) are hardcoded within the `MaxSumSubarrayPageContent` component, similar to the Two Sum example.
