## The Bottleneck Problem

Processing large volumes of log data typically hits a wall at the I/O layer. Most naive implementations spend more time waiting for the disk or the OS to hand over data than they do actually parsing the text.

## Zero-Copy Parsing

One of the most effective strategies for performance is avoiding unnecessary allocations. By using techniques like memory mapping (`mmap`), we can treat a file on disk as if it were in memory, allowing the OS to handle the paging while we parse the data in place.

### Architectural Pillars

*   **Sequential Access**: Modern hardware is optimized for reading data linearly.
*   **Minimal Abstractions**: Every layer of abstraction between the raw bytes and the logic adds overhead.
*   **Parallelism**: Distributing the parsing load across multiple CPU cores without introducing contention.

## Why it Matters

In a production environment, an analyser that takes 10 seconds to run is a tool; one that takes 100 milliseconds is an extension of the developer's thought process. Speed is a feature that enables better debugging and faster iteration.
